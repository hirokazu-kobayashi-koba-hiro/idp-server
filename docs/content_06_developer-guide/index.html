🎯 Control Plane vs Application Plane​

Control Plane（管理API）​

役割: システム・リソースの設定管理

  • URL: /v1/management/...
  • ユーザー: システム管理者・組織管理者
  • 実装: 10フェーズパターン（権限チェック・Audit Log・Dry Run）

Application Plane（OAuth/OIDCフロー）​

役割: 認証・認可フロー実行

  • URL: /{tenant-id}/v1/...
  • ユーザー: エンドユーザー・アプリケーション
  • 実装: シンプルな委譲パターン（権限チェックなし）

🎓 役割別ラーニングパス​

全員共通​

  • 初級（1-2週間） - アーキテクチャ理解・バグ修正

役割別トラック​

初級完了後、役割に応じて選択：

  • Control Plane Track（2-4週間） - 管理API実装者向け

    • システムレベルAPI・組織レベルAPI実装
    • Repository実装・Context Creator実装

  • Application Plane Track（2-4週間） - 認証フロー実装者向け

    • Authorization Flow・Token Endpoint実装
    • 認証インタラクター・Grant Type・Federation実装

  • Full Stack Track（1-2ヶ月） - 両方マスター

    • Control Plane + Application Plane完全習得
    • 統合実装・アーキテクチャ設計
