# VC の歴史と今後の展望

Verifiable Credentials は発展途上の技術です。このドキュメントでは、VC 関連仕様の時系列と今後の流れを整理します。

## 時系列：主要な出来事

### 2012-2016年：前史

| 年 | 出来事 |
|----|--------|
| 2012 | W3C Web Payments Community Group 発足 |
| 2014 | 「Self-Sovereign Identity」の概念が提唱され始める |
| 2016 | W3C Verifiable Claims Working Group 発足 |
| 2016 | Decentralized Identity Foundation (DIF) 設立 |

### 2017-2019年：基盤仕様の策定

| 年 | 出来事 |
|----|--------|
| 2017 | W3C DID Working Group 発足 |
| 2019 | **W3C VC Data Model 1.0** 勧告（W3C Recommendation） |
| 2019 | ISO 18013-5 (mDL) の策定が本格化 |

```
2019年: VC Data Model 1.0 が W3C 勧告に

  これが VC の「公式な誕生」
  - 基本的なデータ構造を定義
  - JSON-LD ベースの表現
  - 署名方式は複数を許容
```

### 2020-2021年：DID と周辺仕様

| 年 | 出来事 |
|----|--------|
| 2020 | COVID-19 パンデミック → ワクチン証明書への関心が高まる |
| 2021 | EU Digital COVID Certificate 運用開始（VC ベースではないが関心を喚起） |
| 2021 | **ISO 18013-5 (mDL)** 発行 |
| 2021 | Apple Wallet で米国一部州の運転免許証対応を発表 |

### 2022年：DID の標準化

| 年 | 出来事 |
|----|--------|
| 2022 | **W3C DID Core 1.0** 勧告 |
| 2022 | OpenID for Verifiable Credentials (OID4VC) の仕様策定が活発化 |
| 2022 | EU eIDAS 2.0 規則案の発表（EU Digital Identity Wallet） |

```
2022年: DID Core が W3C 勧告に

  VC の発行者・保持者を識別する DID が正式に標準化
  - did:web, did:key など複数のメソッド
  - DID Document の構造を定義
```

### 2023年：SD-JWT と OID4VC

| 年 | 出来事 |
|----|--------|
| 2023 | **W3C VC Data Model 2.0** 勧告 |
| 2023 | SD-JWT 仕様が IETF で最終段階に |
| 2023 | OID4VCI / OID4VP 仕様が安定版に |
| 2023 | 米国で複数州が mDL の本格運用を開始 |
| 2023 | Google Wallet で mDL 対応を拡大 |

```
2023年: VC Data Model 2.0 へのアップデート

  主な変更点:
  - JSON-LD 以外のフォーマット（JWT）を正式サポート
  - セキュリティ強化
  - 用語の整理
```

### 2024年：RFC 化と実用化の進展

| 年 | 出来事 |
|----|--------|
| 2024 | **RFC 9901 (SD-JWT)** 発行 |
| 2024 | EU eIDAS 2.0 施行開始（段階的） |
| 2024 | 日本でマイナンバーカードのスマホ搭載が進む |
| 2024 | 各国で mDL / Digital ID の実証実験が拡大 |

```
2024年: SD-JWT が RFC として正式発行

  選択的開示の標準的な方法が確立
  - JWT エコシステムとの親和性
  - 実装が容易
  - VC との組み合わせ（SD-JWT VC）
```

## 地域別の動向

### 欧州連合（EU）

```
eIDAS 2.0 による強制的な普及:

  2024年: 規則施行
    │
    ▼
  2026年: 加盟国は EU Digital Identity Wallet を提供する義務
    │
    ▼
  2027年以降: 公共サービス、銀行、通信事業者などが Wallet 対応必須

  フォーマット: SD-JWT VC + mdoc の両対応が想定される
```

### 米国

```
州単位での mDL 導入:

  2024年時点:
  - 10州以上で mDL が利用可能（または試験運用中）
  - Apple Wallet / Google Wallet が対応
  - TSA（空港セキュリティ）で mDL 受け入れ拡大

  連邦レベルの統一基準はまだない
```

### 日本

```
マイナンバーカードのデジタル化:

  2023年: スマホへの搭載開始（Android）
  2024年: iPhone 対応
  2025年以降: 運転免許証のデジタル化を検討中

  課題:
  - 国際標準（mDL, VC）との整合性
  - 既存の行政システムとの統合
```

### アジア太平洋

| 国・地域 | 状況 |
|---------|------|
| 韓国 | mDL 運用中、政府主導で積極的 |
| オーストラリア | mDL 試験運用中 |
| シンガポール | Singpass での Digital ID 運用中 |

## 今後の展望

### 短期（2025-2026年）

```
予想される動き:

  1. EU Digital Identity Wallet の本格展開
     - 4億人以上の市民に影響
     - 企業の対応が必須に

  2. mDL の国際相互運用
     - 国境を越えた免許証確認
     - ISO 18013-7（オンライン検証）の標準化

  3. SD-JWT VC の普及
     - RFC 化により採用が加速
     - OID4VCI/OID4VP との組み合わせが標準的に

  4. Wallet アプリの競争激化
     - Apple, Google に加え、サードパーティ Wallet
     - 企業向け Wallet ソリューション
```

### 中期（2027-2030年）

```
予想される動き:

  1. 物理的な証明書の置き換え
     - 運転免許証、パスポート、学生証などがデジタル化
     - ただし物理カードも並行して存在

  2. プライバシー技術の進化
     - BBS+ 署名などの Unlinkability 対応技術
     - ゼロ知識証明の実用化

  3. 企業間での VC 活用
     - サプライチェーンでの認証
     - 従業員資格の証明
     - B2B 取引での信頼構築

  4. 規制の整備
     - 各国での法的枠組みの確立
     - 国際的な相互運用の合意
```

### 長期（2030年以降）

```
不確実だが考えられるシナリオ:

  楽観的シナリオ:
    - VC が身分証明の標準に
    - 物理的なカードはほぼ不要に
    - 国際的な相互運用が実現

  現実的シナリオ:
    - 特定のユースケース（政府 ID, 教育）で普及
    - 物理カードとの併用が続く
    - 地域ごとに異なるエコシステム

  悲観的シナリオ:
    - 複雑さや相互運用性の問題で普及が停滞
    - 既存のシステム（パスポート等）が継続
    - VC は限定的なニッチ技術に
```

## 残された課題

### 技術的課題

| 課題 | 状況 |
|------|------|
| Unlinkability | BBS+ 等の研究は進むが、実用レベルには未達 |
| 鍵管理 | 一般ユーザーにとって依然として複雑 |
| 失効確認 | オフラインとリアルタイムのトレードオフ |
| 相互運用性 | SD-JWT VC と mdoc の相互変換など |

### 普及の課題

| 課題 | 状況 |
|------|------|
| 鶏と卵 | 発行者・検証者・ユーザーの同時普及が必要 |
| UX | QR スキャン、承認フローなどの煩雑さ |
| 信頼構築 | 発行者の信頼リスト、ガバナンス |
| 法的整備 | デジタル署名の法的効力、責任分担 |

```
VC の現在地と今後:

  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  2019: VC Data Model 1.0 ─────┐                            │
  │  2021: ISO 18013-5 (mDL) ─────┤                            │
  │  2022: DID Core 1.0 ──────────┼──► 基盤仕様が確立          │
  │  2023: VC Data Model 2.0 ─────┤                            │
  │  2024: RFC 9901 (SD-JWT) ─────┘                            │
  │                                                             │
  │                    ↓                                        │
  │                                                             │
  │  2024-2026: 初期採用期                                      │
  │    - EU eIDAS 2.0 が牽引                                    │
  │    - 政府 ID（mDL）が先行                                   │
  │    - 企業での実証実験                                       │
  │                                                             │
  │                    ↓                                        │
  │                                                             │
  │  2027以降: 普及期（予想）                                   │
  │    - 一般消費者への浸透                                     │
  │    - 物理証明書の代替                                       │
  │    - 国際相互運用                                           │
  │                                                             │
  │  ただし:                                                    │
  │    - 普及のスピードは不確実                                 │
```
